Performance dos algoritmos de ordenação

Análise da complexidade temporal e eficácia dos algoritmos de ordenação.

Análise performática

Análise temporal


Nesta sessão, iremos analisar o desempenho de cada um dos algoritmos previamente estudados: - Bubble Sort - Selection Sort - Insertion Sort - Merge Sort

Iremos analisar quanto tempo cada um dos algoritmos leva para ordenar um arranjo de n elementos, sendo n um parâmetro que irá variar de tamanho, iremos analisar quanto cada algoritmo leva para ordenar de forma crescente o arranjo escolhido e exibir na tabela abaixo, que exibe o tempo, em segundos, em que cada algoritmo leva para ordenar n elementos que aqui serão apresentados na ordem de 8 para aumentar rapidamente a quantidade de elementos no arranjo apresentado:

Algoritmos de ordenação e quanto cada um levou para ordenar elementos na potência de 8

Algoritmo 8¹ 8² 8³ 8⁴ 8⁵
Bubble Sort 0.000021 0.000938 0.033170 2.429335 183.438558
Selection Sort 0.000015 0.000278 0.011274 0.746503 57.731155
Insertion Sort 0.000010 0.000149 0.013343 1.227471 57.685827
Merge Sort 0.000024 0.000306 0.002418 0.019222 0.181762

Podemos analisar, por meio do gráfico, que o método Merge Sort é o mais consistente e eficiente pois independente do tamanho do array informado, seu tempo para ordenar é praticamente o mesmo. Os métodos Insertion Sort e Selection Sort têm desempenho praticamente idêntico e ficam no meio do caminho, pois seus tempos de execução aumentam conforme o tamanho array aumenta, porém não de forma tão acentuada quanto o Bubble Sort, que após atingida uma certa quantidade de elementos no array, seu uso fica praticamente inviável em termos de custo operacional e tempo de execução.
